Introducción

Vamos a trabajar con ensambles de distintas cepas de E. coli hospedero Bovino.

NOTA ⚠️👁️‍🗨️: Para las figuras del curso se utilizaron todos los 110 genomas con hospedero Bovino. Para este tutorial generaremos un subconjunto de 16, ya incluidos genomas de referencia para patotipos EHEC y EPEC.

Para iniciar vamos a descargar los datos disponibles que existan usando datasets de NCBI, lo puedes instalar aquí.

Con el siguiente código vamos a descargar los genomas depositados en NCBI de E. coli

mkdir 1_data
cd 1_data

datasets \
summary genome taxon '562' \
--assembly-level complete\
--assembly-source refseq \
--as-json-lines > Escherichia_coli_562.json
dataformat tsv genome \
--inputfile Escherichia_coli_562.json > Escherichia_coli_562.tsv

Este conjunto de metadatos Escherichia_coli_562.tsv nos servirá para seleccionar los ensambles de genoma que necesitamos, este archivo cuenta con 58533 filas y 174 columnas.

Por ello es necesario utilizar la plataforma o método que más prefieras, y quedarnos con un conjunto más pequeño de genomas para descargar.

Descarga de archivos

Para descargar los archivos vamos a utilizar el siguiente código:

for g in $(cut -f1 Ecoli_bovineHost.tsv | awk 'NR > 1'); do
        if [ -f $g.zip ]; then
            echo $g'.zip exists';
        else
            datasets download genome accession $g \ 
            --include genome --filename $g'.zip';
            sleep 2;
        fi
    done

Descomprimir los archivos

for g in $(cut -f1 Ecoli_bovineHost.tsv | awk 'NR > 1'); do
        unzip $g.zip -d $g;
        mv $g/ncbi_dataset/data/$g/*.fna $g.fna;
        rm -r $g/;
done

rm *.zip

Anotación de genomas

Prokka

Prokka es un anotador rápido y estándar para genomas procariontes (bacterias, arqueas y virus asociados) que identifica características genómicas (CDS, tRNAs, rRNAs, ncRNAs, etc.), las anota con nombres/IDs y genera archivos listos para análisis posteriores o envío a bases públicas.

genomes=$( ls $HOME/2025_Demo_ecoli/1_data/*.fna )

for g in $genomes; do
 prokka \
 $g \
 --cpus 6 \
 --outdir prokka/${id%.fna} \
 --genus Escherichia
done

  • genomes=.../*.fna — crea la lista de archivos FASTA de contigs/consensos que serán anotados.

  • El for g in $genomes; do ... done recorre cada ensamblaje.

  • prokka $g — indica a Prokka el archivo de entrada (.fna/.fasta).

  • --cpus 6 — usa 6 hilos para acelerar pasos paralelizables (e.g., búsquedas).

  • --outdir prokka/${id%.fna} — dirige la salida a una carpeta por genoma (nota: en el snippet que enviaste la variable id no está definida dentro del bucle; ver corrección más abajo).

  • --genus Escherichia — parámetro opcional que ayuda a priorizar nombres/funciones (mejorará la selección de nombres genéricos). (GitHub)

Principales archivos que genera Prokka y para qué sirven

  • <sample>.gff — archivo maestro en formato GFF3 que contiene las características anotadas (coordenadas, tipos, anotaciones). Usarlo para cargar en IGV/Artemis o como entrada a pipelines que esperan GFF. (nf-co.re, stab.st-andrews.ac.uk)

  • <sample>.gbk (o .gbk.gz) — archivo en formato GenBank (útil para visualización/depósito y para herramientas que leen GenBank). (nf-co.re)

  • <sample>.fna — la secuencia de nucleótidos (contigs) usada/normalizada por Prokka (puede incluir cambios en nombres de secuencia). (stab.st-andrews.ac.uk)

  • <sample>.ffn — secuencias de nucleótidos de los genes codificantes (CDS) extraídas. (stab.st-andrews.ac.uk)

  • <sample>.faa — traducciones (secuencias proteicas) de los CDS; útil para búsquedas funcionales (BLAST/DIAMOND), clustering o análisis de pangenoma. (stab.st-andrews.ac.uk)

  • <sample>.tbl — tabla sencilla con información de anotación (útil para envío a GenBank/ENA en algunos workflows). (HPC NIH)

  • <sample>.sqn (opcional) — formato Sequin para enviar a NCBI (si se generan). (HPC NIH)

  • PROKKA_*.log / prokka.err — logs que detallan programas internos y pasos realizados (útil para depuración).

Notas prácticas y recomendaciones para Prokka

  • Si tu entrada es multi-FASTA con múltiples contigs, Prokka generará un GenBank multi-registro con uno por contig.

  • Ajusta --kingdom, --genus, --species, --strain para mejorar la nomenclatura.

  • Prokka usa varias bases (p. ej. UniProt/Swiss-Prot) y heurísticas; para anotaciones más «exhaustivas» posteriores se pueden re-analizar las proteínas (.faa) con herramientas especializadas. (GitHub)

Bakta

Bakta es un anotador diseñado para obtener anotaciones amplias, normalizadas y «dbxref-ricas» (múltiples referencias cruzadas) en genomas bacterianos y plásmidos. Está pensado para producir salidas estandarizadas (incluyendo JSON legible por máquina) y sORFs (small ORFs) además de las características estándar (CDS, tRNA, rRNA, ncRNA, CRISPR, etc.). Usa una base de datos local y herramientas rápidas (p. ej. DIAMOND) para hacer búsquedas de homología. (Bakta Documentation, PMC)

source $HOME/bin/bakta-env/bin/activate
Muestra el código 
genomes=$(ls $HOME/2025_Demo_ecoli/1_data)

experiment_file="$HOME/2025_Demo_ecoli/Experiment_Design.tsv"
genomes=$(ls $HOME/2025_Demo_ecoli/1_data)

experiment_file="$HOME/2025_Demo_ecoli/Experiment_Design.tsv"

for g in $genomes; do
    id=$(basename "$g" | awk -F'/' '{print $NF}' | sed 's/.fna$//')

    strain=$(awk -F'\t' -v id="$id" '$1 == id {print $3}' "$experiment_file")

    bakta \
    --db $HOME/2025_Demo_ecoli/db \
    --output $HOME/2025_Demo_ecoli/2_annotation/bakta/${id} \
    --genus Escherichia \
    --strain "$strain" \
    --threads 6 \
    --force $HOME/1_data/${g}
done

  • source .../bin/activate — activa el entorno (p. ej. virtualenv/conda) donde está Bakta.

  • --db $HOME/.../db — ubicación de la base de datos compacta de Bakta (necesaria; puede ocupar decenas de GB). (Bakta Documentation)

  • --output ... — carpeta por genoma para guardar resultados.

  • --genus / --strain — ayudan en la anotación y en la generación de nombres.

  • --threads 6 — número de hilos para paralelizar búsquedas.

  • --force — sobrescribir salidas existentes.

  • Bakta internamente usa búsquedas con DIAMOND/BLAST y una base SQLite con digest y anotaciones preasignadas (esto acelera y estandariza el proceso). (Bakta Documentation, Bactopia)

Precauciones de recursos ⚠️

Bakta requiere memoria y disco para su base y para los pasos intermedios; es habitual necesitar varios GB de RAM (tu nota de 8 GB es razonable como mínimo) y tiempo cuando se procesan muchos genomas. Si tienes >10 genomas, espera que el tiempo total aumente y planifica en un clúster o máquina con suficientes recursos. (Bactopia, software.cqls.oregonstate.edu)

Archivos y salidas importantes que produce Bakta

  • annotation.gff / <id>.gff — GFF3 con anotaciones (coordenadas y atributos). (Bakta Documentation)

  • <id>.gbk — GenBank (cuando se solicita o se genera). (Bakta Documentation)

  • <id>.faa, <id>.ffn — proteínas y secuencias nucleotídicas de CDS (similares a Prokka). (Bakta Documentation)

  • report.json — salida JSON rica en metadatos y dbxrefs (útil para pipelines automatizados). Bakta hace énfasis en producir JSON estandarizado que puede alimentar otras herramientas. (Bakta Documentation, GitHub)

  • bakta.log / registros — salida estándar y errores para depuración. (Bakta Documentation)

Ventajas frente a Prokka

  • Bakta genera metadatos más ricos / dbxrefs y JSON preparado para downstream automático.

  • Usa una base precomprimida con mapeos a UniProt (y otras colecciones), lo que suele producir IDs más estandarizados.

  • Prokka es muy rápido y ampliamente usado; Bakta tiende a ser más exhaustivo y «estandarizado» pero con requisitos mayores de disco/DB. Para workflows, muchos usuarios ejecutan Prokka para anotaciones rápidas y Bakta cuando quieren anotaciones más normalizadas y ricas en dbxrefs. (PMC, Bactopia)

Consejos para decidir cuál usar en tu pipeline

  • Si quieres velocidad y una anotación estándar rápida para muchas muestras: usa Prokka y luego procesa los .faa con herramientas funcionales (eg. EggNOG, InterProScan) según necesites. (GitHub)

  • Si quieres anotaciones más ricas, normalizadas y con dbxrefs/JSON para downstream automático (p. ej. bases de datos locales, comparaciones taxonómicas o reporte estandarizado), usa Bakta (pero planifica la DB y recursos). (Bakta Documentation, PMC)

Pangenoma

Vamos a utilizar Roary, lo puedes descargar aquí.

Preparamos los archivos .gff obtenidos de la anotación con Prokka o Bakta para poder realizar el análisis de pangenoma.

DIR_ORIGEN="$HOME/2025_Demo_ecoli/2_annotation/prokka"
DIR_DESTINO="$HOME/2025_Demo_ecoli/3_pangenome/data_genomes"
for carpeta in "$DIR_ORIGEN"/*; do
        if [[ -d "$carpeta" ]]; then
            nombre_carpeta=$(basename "$carpeta")

            # Buscar el primer archivo .gff o .gff3 en la carpeta 
            archivo_gff=$(find "$carpeta" -maxdepth 1 -type f \( -iname "*.gff" -o -iname "*.gff3" \) -print -quit || true)

            if [[ -n "$archivo_gff" ]]; then
                destino="$DIR_DESTINO/${nombre_carpeta}.gff"
                cp -f -- "$archivo_gff" "$destino"
                echo "Archivo $archivo_gff copiado como $destino"
            else
                echo "No se encontró archivo .gff/.gff3 en $carpeta"
            fi
        fi
    done
DIR_ORIGEN="$HOME/2025_Demo_ecoli/2_annotation/prokka"
DIR_DESTINO="$HOME/2025_Demo_ecoli/3_pangenome/data_genomes"
